# Campus Plans - Планы корпусов

## Поддерживаемые форматы

Система поддерживает загрузку планов корпусов в двух форматах:
- **PNG** (приоритетный формат, рекомендуется)
- **SVG** (используется как fallback, если PNG отсутствует)

## Именование файлов

Файлы должны быть названы по шаблону: `{корпус}-{этаж}.{расширение}`

Примеры:
- `1-0.png` - план корпуса 1, этаж 0 (цокольный)
- `1-1.png` - план корпуса 1, этаж 1
- `2-3.svg` - план корпуса 2, этаж 3

## Как загрузить PNG планы

### Вариант 1: Замена существующих SVG файлов на PNG

1. Конвертируйте ваши SVG файлы в PNG формат
2. Сохраните PNG файлы с тем же именованием (например, `1-0.svg` → `1-0.png`)
3. Поместите PNG файлы в эту директорию: `src/main/resources/static/campus-plans/`
4. Система автоматически использует PNG файлы с приоритетом над SVG

### Вариант 2: Добавление новых планов

1. Создайте PNG файл с планом корпуса
2. Назовите файл согласно шаблону: `{номер_корпуса}-{номер_этажа}.png`
3. Скопируйте файл в `src/main/resources/static/campus-plans/`
4. Перезапустите приложение

## Приоритет форматов

Если для одного и того же корпуса и этажа существуют оба файла (PNG и SVG), система **автоматически выберет PNG** файл для отображения.

Пример:
- Существуют файлы: `1-1.png` и `1-1.svg`
- Отобразится: `1-1.png` ✓

## Рекомендации

- **Разрешение**: рекомендуется использовать PNG файлы с разрешением не менее 1920x1080px для качественного отображения
- **Размер файла**: оптимизируйте PNG файлы для веба (рекомендуется не более 500KB на файл)
- **Формат**: используйте PNG-24 для лучшего качества изображения
